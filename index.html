Macroeconomic Research Tool

Go to GitHub Project Page >>>

Though I love virtually all topics in economics, recent events have led me to take a particularly strong interest in US Dollar inflation. Starting as a Project for my data analytics bootcamp and continuing as an independent project long after, I wanted to create a research dashboard for economists like me to examine macroeconomic trends and to directly examine relationships between economic indicators.

The user of the applciation can choose up to 5 data series to visualize on the line chart. Additionally, the user may choose a background for the line chart that displays recessions, presidential administrations, or major world events. The summary statistics tables to the right of the line chart give the user a handy way to compare the long-term changes and trends in the different series, and they re-calculate when the user zooms in in a particular date range or changes which series are displayed on the line chart. There are a total of 123 different series to choose from, and the data is mostly sourced from the St. Louis Federal Reserve's FRED database. A few of the series are sourced from the Wall Street Journal.


If the user wants to take a closer look at the relationship between two series, I have included a scatter plot and a gauge to display the correlation between the two variables. In addition to the 123 main series, the user may choose to examine the correlation between the "change" or the "percent change" between the variables. Further, since some economic trends don't influence other indicators for several days, weeks or months after changing, the user may choose to lag the variable by up to 10 of the variable's reporting intervals (days, weeks, months, quarters, years), as well as filter to a precise date range.

The Lag feature of this portion of the application was one of the most difficult things to build. Because the data is reported on all different intervals, I had to pick a method of getting the data to match up if the user wanted to compare, for example, a weekly series with a quarterly series. The first version of this application takes the "lowest common denominator" approach, where a weekly series is reduced to a quarterly interval to match the interval of the other series. In a future version of the application, I plan to use a neural network for imputation of the series with the wider interval (essentially, to predict what weekly values of a quarterly series would be).


Finally, I have included three models for predicting the level of inflation as measured by the Consumer Price Index (CPI), Personal Consumption Expenditures (PCE), and the GDP Deflator based on user inputs. It is worth noting that these models are currently DEEPLY FLAWED. They are full of omitted variable bias, biases due to correlated input variables, and none of the data was properly scaled prior to building the models. I have chosen to leave them in the application despite these flaws because I like the user interface, and I will be replacing them with much better models in the near future. I have already created some of the models in a separate-but-related project (see below).

ARCHITECTURE: I am the sole author of this application. I don't pretend to be an expert web designer, but I can do enough in HTML (Bootstrap) and CSS to get by. The application is primarily built using a python Flask file in conjunction with a JavaScript (D3.js) file, an HTML file, and a CSS file. The data is stored in a PostgreSQL database. I invite anyone interested to take a look at the code on my GitHub page.

Modeling US Dollar Inflation

Go to GitHub Project Page >>>

I am extremely excited about this project! Initially intended to be included in my Macroeconomic Research Tool project (above), I decide to separate this to make it its own project because I want to keep working on the various models for the foreseeable future. This entire project is done in Jupyter Notebook, though I plan to integrate my favorite models into the Macroeconomic Research Tool.

The first model was intended to give me a baseline for what the most basic neural network would look like with minimal data preprocessing. It was built using a Scikit-Learn MLP neural network. It has a single hidden layer with 100 nodes, and the training data is unscaled. I used data from the end of Bretton Woods in 1971 through the beginning of the COVID-19 pandemic in 2021. The model did a poor job predicting the value for CPI based on post-pandemic inputs.

Model 2 switches from Scikit-Learn to Keras and Tensorflow. Similarly, it is a very basic neural network with a single hidden layer and 100 nodes, and it too uses unscaled data. It does a slightly better job of predicting CPI, though it too is intended to be just a baseline.

Model 3 also uses Keras, but this version uses data scaled using Scikit-Learn's MinMaxScaler function to scale all values to betweeen 0 and 1. The model has a single hidden layer with 100 nodes, and it propagates through 100 epochs. The improvement over the previous models is substantial, through the model is still not particularly accurate.


Model 4 is the best model yet! I increased the input variables from 8 to 12, there are two hidden layers instead of one (still with 100 nodes each), and I again propagated through 100 epochs. Like the previous model, I again used Scikit-Learn's MinMaxScaler function to scale all variables between 0 and 1. This model (an all above models) are trained on data starting 10/1/1971 at the end of Bretton Woods, and ending on 1/1/2020 before the COVID-19 pandemic. Incredibly, the model predicted CPI for April of 2021 with perfect precision despite many of the input variables breaking from their long-term trends once the pandemic began.

Model 5 is a work in progress. I intend for it to be a custom model loosely based on the Quantity Theory of Money. I will model each of the four variables in the QTM separately, then create an ensemble model to predict the price level. I believe the QTM is generally accurate in the long run, but there are short-term pressures that influence each of the variables in MV = PY.
